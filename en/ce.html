Context Engineering

Designing context for LLMs

Context Engineering refers to the systematic design of information context provided to an LLM. The context window is limited – what you include and how you arrange it determines the quality of the output. More is not automatically better.

Theory Exercises Deep Dive Resources

Theory

Key Points

  • Context window: GPT-4o 128K, Claude 200K, Gemini 1M tokens
  • More context is not automatically better – Lost in the Middle problem
  • Proven structure: Role → Task → Information → Constraints → Format
  • Compression: Relevant excerpts instead of entire files
  • Examples are often more effective than long descriptions
  • Make project conventions explicit

Concepts

RAG
Retrieval-Augmented Generation – retrieving relevant information from external sources and providing it to the LLM
Lost in the Middle
LLMs lose information in the middle of long contexts – place important content at beginning/end
Context Structure
Proven arrangement: Role → Task → Information → Constraints → Format
Context Window
The maximum amount of text (in tokens) that an LLM can process simultaneously
Distillation
Compressing complex information to its essentials

The 5-Part Context Structure

context_template.txt 
# 1. ROLE
You are an experienced Python developer focusing on data processing.

# 2. TASK
Write a script that merges Excel files.

# 3. INFORMATION
Input files: 50 .xlsx in folder /data/
Columns: ID, Name, Date, Value
Encoding: UTF-8

# 4. CONSTRAINTS
- No external dependencies except openpyxl
- Skip faulty rows, don't abort
- Log to file, not just console

# 5. FORMAT
- Comments in English
- Functions with docstrings
- Single script, no modules

Exercises

CE-1: Structuring Context

Build a prompt using the proven context structure.

Learning Objectives:
  • Apply the 5-part context structure
  • Separate relevant from irrelevant information
  • Compare effectiveness

Compare these two prompts:

context_comparison.txt 
# WITHOUT context structure:
"Make me a script for my Excel files"

# WITH context structure:
"""
You help me write a Python script.

Task: Read all Excel files in a folder
and merge them into a single CSV.

Example data (first file):
| Inv-Nr | Description | Dating |
| 001    | Vase        | 1820   |

Constraints:
- All files have identical column structure
- Encoding: UTF-8
- Only .xlsx, no .xls

Output: A single script with comments.
"""

Task: Test both prompts with an LLM. Document the differences in results.

Reflection:
  • What clarifying questions did the LLM ask with the first prompt?
  • How much more precise was the result with the second prompt?
  • What information did you add in the structured prompt?

CE-2: Distilling Knowledge

Prepare a complex standard so it fits in the context window.

Learning Objectives:
  • Identify relevant information
  • Compress context in a structured way
  • Test effectiveness

Task: Visit the Darwin Core documentation (dwc.tdwg.org) and distill the relevant fields for a museum collection. Create a compact reference document with field name, definition, and example value.

Reflection:
  • How much shorter did the document become?
  • What information did you omit?
  • Did the LLM work correctly with the distilled context?

Deep Dive

Lost in the Middle Problem

Research shows: LLMs process information at the beginning and end of context better than in the middle. Consequences:

  • Important information at the beginning or end
  • Examples at the beginning, constraints at the end
  • For long contexts: repeat summary at the end

When More Context Hurts

  • Irrelevant details distract and confuse
  • Contradictory information leads to inconsistent outputs
  • Too many examples can lead to overfitting on examples
  • Outdated information overwrites current knowledge

Context Compression Techniques

Summarization
Condense long documents into 1-2 paragraphs
Relevant Excerpts
Only the parts of a document relevant to the task
Structured Formats
JSON/YAML instead of prose – more compact and precise
Example Instead of Description
A good example says more than three paragraphs of explanation
