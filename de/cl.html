Code Literacy

Generierten Code verstehen

Code Literacy bezeichnet die Fähigkeit, Code zu lesen und zu verstehen. Das ist eine eigenständige Kompetenz, die dem Schreiben vorausgeht. Man muss nicht selbst Code schreiben können, um LLM-generierten Code zu bewerten. Die Skill-Hierarchie: Run → Trace → Explain → Write.

Theorie Übungen Vertiefung Ressourcen

Theorie

Kernpunkte

  • Lesen und Schreiben sind getrennte Kompetenzen (Lopez et al., 2008)
  • Skill-Hierarchie: Run → Trace → Explain → Write
  • Auch Experten vermeiden tiefes Verstehen wenn möglich – partielles Verstehen ist normal
  • 39% der Praktiker finden LLM-Code lesbarer als menschlichen Code
  • Zwei mentale Modelle: prozedural ("Was passiert der Reihe nach?") und funktional ("Was ist das Ziel?")
  • Sie müssen nicht alles verstehen – nur genug, um zu prüfen

Konzepte

Skill-Hierarchie
Run → Trace → Explain → Write: Lesen und Nachvollziehen gehen dem Schreiben voraus
Prozedurales Modell
"Was passiert der Reihe nach?" – Kontrollfluss, Zeile für Zeile nachvollziehen
Funktionales Modell
"Was ist das Ziel?" – Zweck verstehen, Eingabe/Ausgabe identifizieren
Delocalized Plans
Code einer Intention ist oft über mehrere Stellen verstreut – Navigation erforderlich
Kontrollstruktur
Anweisungen, die den Ablauf steuern (if, for, while)
Bibliothek
Sammlung von vorgefertigtem Code für bestimmte Aufgaben

Die Skill-Hierarchie

Run – Code ausführen können
  • Python installieren und starten
  • Ein Skript ausführen
  • Fehlermeldungen erkennen (nicht verstehen)
Trace – Ausführung nachvollziehen
  • Welche Zeile wird gerade ausgeführt?
  • Was steht in welcher Variable?
  • Welcher Pfad wird bei einer Bedingung genommen?
Explain – Zweck erklären
  • Was macht dieser Codeblock?
  • Warum steht das hier?
  • Was passiert bei dieser Eingabe?
Write – Selbst schreiben
  • Für AI Coding Literacy nicht zwingend erforderlich
  • Aber: Kleine Anpassungen können notwendig sein

Übungen

CL-1: Variablen und Datentypen

Die Grundbausteine von Code verstehen.

Lernziele:
  • Variablen als benannte Speicherplätze verstehen
  • Grundlegende Datentypen unterscheiden
  • Zuweisungen lesen können
variablen.py 
# Variablen sind benannte Speicherplätze
name = "Vase"           # String (Text)
inventar_nr = 1234      # Integer (Ganzzahl)
preis = 45.50           # Float (Dezimalzahl)
ist_ausgestellt = True  # Boolean (Wahr/Falsch)

# Listen speichern mehrere Werte
materialien = ["Keramik", "Glasur", "Gold"]

# Dictionaries speichern Schlüssel-Wert-Paare
objekt = {
    "name": "Vase",
    "jahr": 1820,
    "standort": "Raum 3"
}

Aufgabe: Lesen Sie den Code. Was ist der Wert von `objekt["jahr"]`? Welchen Typ hat `ist_ausgestellt`?

Reflexion:
  • Welche Datentypen kommen in Ihren Daten vor?
  • Wann würden Sie eine Liste vs. ein Dictionary verwenden?

CL-2: Kontrollstrukturen lesen

Verzweigungen und Schleifen verstehen.

Lernziele:
  • if/else-Verzweigungen nachvollziehen
  • for-Schleifen verstehen
  • Einrückung als Strukturmerkmal erkennen
kontrollstrukturen.py 
dateien = ["bild1.jpg", "dokument.pdf", "bild2.png", "text.txt"]

# Für jede Datei in der Liste
for datei in dateien:
    # Prüfe die Endung
    if datei.endswith(".jpg") or datei.endswith(".png"):
        print(f"Bild gefunden: {datei}")
    elif datei.endswith(".pdf"):
        print(f"PDF gefunden: {datei}")
    else:
        print(f"Andere Datei: {datei}")

Aufgabe: Welche Ausgabe erzeugt dieser Code? Gehen Sie Zeile für Zeile durch.

Reflexion:
  • Was passiert bei einer leeren Liste?
  • Was passiert bei einer Datei namens "BILD.JPG" (Großbuchstaben)?

CL-3: Funktionen verstehen

Wiederverwendbare Codeblöcke erkennen.

Lernziele:
  • Funktionsdefinitionen erkennen
  • Parameter und Rückgabewerte verstehen
  • Funktionsaufrufe nachvollziehen
funktionen.py 
def formatiere_name(vorname, nachname, titel=None):
    """Formatiert einen Namen für die Anzeige."""
    if titel:
        return f"{titel} {vorname} {nachname}"
    else:
        return f"{vorname} {nachname}"

# Funktionsaufrufe
name1 = formatiere_name("Maria", "Müller")
name2 = formatiere_name("Hans", "Schmidt", "Dr.")

print(name1)  # → Maria Müller
print(name2)  # → Dr. Hans Schmidt

Aufgabe: Was gibt `formatiere_name("Anna", "Weber", "Prof.")` zurück?

Vertiefung

Lesestrategien für Code

Top-Down
Erst die Struktur verstehen, dann die Details. Was sind die Hauptfunktionen?
Bottom-Up
Bei Fehlern: Von der Fehlerstelle zurückverfolgen. Was kam davor?
Dokumentation zuerst
Kommentare, Docstrings, README lesen bevor man den Code analysiert.
LLM als Erklärer
"Erkläre mir diesen Code" – nutzen Sie das LLM als Tutor.

Was Sie nicht verstehen müssen

  • Jede Bibliotheks-Funktion – `pandas.read_csv()` liest CSV, Details egal
  • Komplexe Algorithmen – Solange Sie wissen, was rein und raus geht
  • Optimierungen – Ob es elegant ist, ist für die Funktion irrelevant
  • Syntaktischen Zucker – List Comprehensions etc. sind Abkürzungen

Red Flags beim Lesen

Diese Muster sollten Sie aufmerksam machen:

  • Keine Fehlerbehandlung – Was passiert wenn die Datei nicht existiert?
  • Hardcoded Werte – `/home/user/data/` funktioniert nur auf einem Rechner
  • Keine Kommentare – Bei komplexer Logik ein Warnsignal
  • Sehr lange Funktionen – Schwer zu verstehen, oft Fehleranfällig
