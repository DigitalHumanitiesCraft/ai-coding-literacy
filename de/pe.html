Prompt Engineering

Effektive Prompts entwickeln

Prompt Engineering bezeichnet die Entwicklung und Optimierung von Eingabeaufforderungen. Ein Prompt ist mehr als eine Frage – die Art, wie man fragt, bestimmt die Art der Antwort. Schulhoff et al. (2024) dokumentieren 58 verschiedene Prompting-Techniken. Für die meisten Anwendungen genügen wenige Kernstrategien.

Theorie Übungen Vertiefung Ressourcen

Theorie

Kernpunkte

  • Schulhoff et al. dokumentieren 58 Prompting-Techniken – für die Praxis genügen wenige
  • Zero-Shot: Direkte Anweisung ohne Beispiele – für einfache, eindeutige Aufgaben
  • Few-Shot: 2-5 Beispiele zeigen Format und Umgang mit Sonderfällen
  • Chain-of-Thought: "Denke Schritt für Schritt" – für komplexe Probleme
  • Iterative Verfeinerung: Prompt → Output → Problem → Anpassung → wiederholen
  • Ausgabeformat explizit spezifizieren

Konzepte

Zero-Shot
Direkte Anweisung ohne Beispiele – funktioniert bei einfachen, eindeutigen Aufgaben
Few-Shot
2-5 Beispiele im Prompt zeigen das gewünschte Format und den Umgang mit Sonderfällen
Chain-of-Thought
"Denke Schritt für Schritt" – verbessert Leistung bei komplexen, mehrstufigen Problemen
Role Prompting
"Du bist ein erfahrener..." – wenn spezifische Expertise oder Perspektive wichtig ist
Iteration
Prompt → Output prüfen → Problem identifizieren → Prompt anpassen → wiederholen

Wann welche Technik?

Zero-Shot – Wenn die Aufgabe eindeutig ist:
  • "Übersetze diesen Text ins Englische"
  • "Formatiere diese Daten als JSON"
  • "Erkläre diesen Code"
Few-Shot – Wenn das Format wichtig ist:
  • Spezifische Ausgabeformate
  • Umgang mit Sonderfällen demonstrieren
  • Domänenspezifische Konventionen
Chain-of-Thought – Bei komplexer Logik:
  • Mehrstufige Berechnungen
  • Logische Schlussfolgerungen
  • Debugging komplexer Probleme

Übungen

PE-1: Zero-Shot vs. Few-Shot

Den Unterschied zwischen direkter Anweisung und Beispiel-basiertem Prompting verstehen.

Lernziele:
  • Zero-Shot und Few-Shot unterscheiden
  • Erkennen, wann Beispiele helfen
  • Beispiele effektiv formulieren
prompting_strategien.txt 
# ZERO-SHOT:
"Extrahiere das Datum aus diesem Text:
'Das Gemälde wurde am 15. März 1823 erworben.'"

# FEW-SHOT:
"Extrahiere das Datum aus Texten. Formatiere als YYYY-MM-DD.

Text: 'Erworben im Januar 1820' -> 1820-01-00
Text: 'Datiert auf den 3.5.1899' -> 1899-05-03
Text: 'Das Gemälde wurde am 15. März 1823 erworben.' ->"

Aufgabe: Testen Sie beide Strategien. Wann liefert Few-Shot bessere Ergebnisse? Wann genügt Zero-Shot?

Reflexion:
  • Wie unterschieden sich die Ausgaben?
  • Welche Sonderfälle haben die Beispiele abgedeckt?
  • Wann würden Sie Few-Shot in Ihrer Arbeit einsetzen?

PE-2: Iteratives Prompting

Durch Dialog schrittweise zum gewünschten Ergebnis.

Lernziele:
  • Feedback präzise formulieren
  • Typische Anpassungen kennen
  • Wissen, wann man neu anfangen sollte
iteration.txt 
# Iteration 1:
"Schreibe ein Skript, das Bilder verkleinert."
# Problem: Unklar welche Größe, welches Format, welcher Ordner

# Iteration 2:
"Schreibe ein Python-Skript mit Pillow.
Eingabe: Ordnerpfad als Argument
Aufgabe: Alle .jpg auf max. 1200px Breite skalieren
Ausgabe: Neue Dateien mit Suffix '_web'"
# Problem: Überschreibt Originale

# Iteration 3:
"... (wie oben)
Constraint: Originaldateien nicht überschreiben
Constraint: Ausgabe in Unterordner 'web/'"

Aufgabe: Starten Sie mit einem einfachen Prompt für Ihr Projekt. Dokumentieren Sie 3 Iterationen: Problem → Anpassung → Ergebnis.

Reflexion:
  • Wie viele Iterationen brauchten Sie?
  • Welche Art von Feedback war am effektivsten?
  • Wann hätten Sie besser neu angefangen?

Vertiefung

Chain-of-Thought in der Praxis

Bei komplexen Problemen hilft es, das LLM zum schrittweisen Denken anzuleiten:

chain_of_thought.txt 
# Ohne Chain-of-Thought:
"Berechne die Gesamtkosten für 47 Bücher à 23,50€ mit 19% MwSt."

# Mit Chain-of-Thought:
"Berechne die Gesamtkosten für 47 Bücher à 23,50€ mit 19% MwSt.
Denke Schritt für Schritt:
1. Berechne den Nettopreis
2. Berechne die MwSt
3. Berechne den Bruttopreis
Zeige jeden Schritt."

Typische Iterationsmuster

"Zu allgemein"
→ Konkrete Beispiele hinzufügen
"Falsches Format"
→ Ausgabeformat explizit spezifizieren oder Beispiel geben
"Falsche Annahmen"
→ Constraints und Abgrenzungen explizit machen
"Zu komplex"
→ Problem zerlegen, einzeln lösen
"Inkonsistent"
→ Neuer Prompt, von vorne anfangen

Anti-Patterns vermeiden

  • Zu viele Aufgaben – Ein Prompt, ein Ziel
  • Vage Formulierungen – "gut", "schön", "passend" → konkretisieren
  • Widersprüchliche Anweisungen – "kurz aber ausführlich"
  • Übermäßiges Role-Prompting – "Du bist der beste Experte der Welt" hilft nicht
